# 06.6 - Container Mapping System

## Containers and Container Grids

**If [[Entity]]** `K- or ven-` #prop_isContainer `True`

**Add** #propContainer_gridMap to define position(s) of another [[Entity]] contained within it.
These are called "***Container Grid Maps***".

#prop_worldScale is defined per [[Entity]] and controls the #propContainer_contentsScale --

An [[Entity]] of a larger [[worldScale]] cannot be contained within a smaller [[worldScale]] [[Entity]] unless #propContainer_isTardis --

#propContainer_isTardis is a `boolean` setting which defines if container can hold things larger in [[worldScale]] than itself.

When an [[Entity]] is a [[Container]], that which is contained within it will be placed in relational Space via the #propContainer_gridMap.

---
### Example: The Tavern
The Tavern is an [[Entity]] set as a [[Container]]. It has 2 floors and contains several Rooms and Zones inside of Rooms.

#### Zones
Assuming the following is contained within the Tavern itself:
```
ven.Tavern/ven.TavernFloor1 contains(
	ven.Fireplace004(2x2),
	ven.Bar001(3x1),
	ven.Door06>ven.Bathroom073(1x1),
	ven.Stairs001>ven.Floor2(1x2),
	ven.Door002>ven.ForestPlot045(1x1)
)
```
Then EVE stores a Map Grid related to the position of these entities within the container.
#### Container Layouts
```
ven.Fireplace004
ven.Stairs001>ven.Floor2
ven.Door06>ven.Bathroom073 
ven.Door002>ven.ForestPlot045

```
