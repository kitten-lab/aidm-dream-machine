# ADM — Core Systems

ADM is composed of several interacting layers that together generate, track, and evolve narrative worlds.

## Intelligences

ADM operates through multiple types of intelligences:

- **System Intelligences (“system girls”)**  
  Responsible for data management, retrieval, and structural operations.  
  They interface with the underlying data systems and ensure that information about the world is accessible and consistent.

- **ADM (Narrative Intelligence)**  
  The central narrative driver.  
  ADM interprets the state of the world and directs narrative progression, coordinating interactions between entities, players, and systems.

- **Agents (“amen”)**  
  Each player is paired with an agent that operates within the ADM layer.  
  These agents participate in narrative flow, mediate interactions, and help carry forward the player’s presence within the system.

---

## Vens (Entity System)

Everything within ADM is represented as a **Ven**.

A Ven is any discrete concept that exists within the world, including:
- characters  
- objects  
- environments  
- abstract entities  

The system treats all elements uniformly as Vens, allowing consistent tracking and interaction.

Vens are managed through a central system that:
- stores metadata about each Ven  
- tracks state and history  
- records interactions and changes over time  

This allows entities to maintain continuity, memory, and identity within the world.

---

## Human Roles

- **Player**  
  Participates directly in the world, interacting with Vens and influencing narrative outcomes.

- **Creator**  
  Defines structures, systems, and rules that shape how the world operates and evolves.

---

Together, these components form a system where:
- data is structured and accessible (System Intelligences)  
- narrative is driven and coordinated (ADM + agents)  
- all elements are consistently represented (Vens)  
- humans participate both as actors and designers
